# T082 deterministic private recommendation delivery

Nutrition Overview now derives private recipe recommendations only for a signed principal holding profile, diary and goal-management rights. The service exposes recurring gaps only after at least three observed days and nutrient-specific average completeness of at least 0.5. Planned gaps require an explicit plan whose relevant meals all use current calculations, meet completeness and confidence thresholds, and contain the target plus every configured limit nutrient. Missing evidence stays unknown and suppresses advice rather than becoming zero.

Candidates use the latest calculation for the current recipe revision and deterministic lexicographic ordering. Low-quality or stale candidates, unresolved allergy evidence, exact allergen conflicts, incomplete exclusion evidence, exact excluded dietary tags and candidates that exceed—or cannot prove compliance with—configured limits are excluded. Explanations name the calculated nutrient amount, gap coverage, calculation quality, Pantry availability, expiring matched products, exact shortages and unknown Pantry reasons without treating stock, planned food or preparation as consumption.

The additive `0023_nutrition_insight_feedback.sql` migration and matching schema persist append-only private dismissed/helpful/not-helpful feedback bound to a canonical evidence-versioned recommendation key. Exact retries are idempotent, stale supersession conflicts, and authorization is rechecked. Recommendation rendering and feedback never mutate Pantry, plans, intake or shopping. A grocery action appears only for an exact shortage and requires the user to choose a list and submit that specific item through the existing trusted-origin shopping mutation.

# T087 private settings delivery audit

## Decision: approved

T086 is approved. The Settings tab is offered only when the selected accessible-profile summary grants `canManageProfile`; the page then calls `getPrivateNutritionProfile` for that exact profile, which reauthorizes `manage_profile`. A non-manager receives neither the navigation item nor the private settings payload. The client sends the complete existing profile shape with the current server version to the unchanged trusted-origin PATCH route, so strict validation and conflict responses remain authoritative.

The form covers every already persisted field. Explanation text precedes the date/body/source-sex/activity/life-stage controls, identifies them as optional and explicitly entered, and keeps manual goals independent. Metric and imperial labels change with the chosen system, deterministic constants convert input to canonical centimeters/kilograms, and focused tests cover the canonical values plus the full optimistic request shape. Privacy choices state both comparison disclosure and the separate need for explicit authorization. No browser persistence, inferred fields or expanded authority was introduced.

Estimator controls accurately describe their limitation: they save consent and inputs but do not calculate, create or replace a goal. This remains a real product gap rather than a hidden claim. Full unit/integration/lint/type/build evidence passed after one unrelated transient Windows backup-file rename retry. Responsive two-column-to-one-column CSS and semantic fieldsets/labels/status messaging provide a sound automated accessibility baseline; rendered keyboard/screen-reader/mobile evidence remains for the final workflow phase.

The next exact task is a read-only authoritative-source Scout. It must verify whether a bounded NASEM 2023 Estimated Energy Requirement implementation is possible from primary published equations using the current explicitly consented profile fields; capture exact age/life-stage/sex-category applicability, coefficients, physical-activity categories, units, dates/version and limitations; identify missing inputs/schema; and propose no Worker unless every numeric term is traceable. It must preserve manual goals, avoid FDA-DV personalization and avoid medical/weight-loss claims.

The exact GoalBuddy Judge exceeded the single 30-second wait and was interrupted. The PM performed the same read-only audit as permitted fallback.

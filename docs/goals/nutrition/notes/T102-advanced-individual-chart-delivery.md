# T102 bounded advanced individual-chart delivery

Delivered a server-only individual chart workspace with a fixed maximum of five database queries and accessible Trends panels for the remaining non-weight chart families.

- A shared server access context performs one profile and one latest-permission read, requires `view_diary`, and reports effective goal-management capability without exposing private profile inputs. The goal query runs only when that same server context authorizes it; diary-only viewers receive `goalContext: unavailable` and no goal rows.
- One range-indexed intake/value query uses a correlated newer-revision exclusion before state/date acceptance. A newer deletion or move outside the visible range cannot resurrect an old revision. Exact local-date membership follows a conservative UTC envelope. Values are left-joined in one query, so entries with missing selected nutrients remain visible to evidence completeness.
- One plan query joins latest planned/served allocations to active meal-plan rows and the latest calculation for the recipe's current revision, then bulk-loads selected nutrient values. Skipped/cancelled plan rows, superseded allocations, ambiguous/missing servings and unavailable calculations are never converted to zero. The query is skipped when planned chart presentation is disabled.
- Historical goals are chosen per series and day by highest applicable revision, then that revision's active/paused/archived state. Future revisions do not rewrite earlier days; pause/archive revisions suppress older active values from their effective date.
- Server datasets preserve `null` missing calories, trailing seven-calendar-day averages over present values only, macro grams and calculated-energy percentages only with all required macros, semantic heatmap evidence/goal states, separate planned and consumed values, immutable snapshot source rankings, and strict missing/partial/fully-documented record-evidence states. Copy explicitly says record completeness does not prove every meal was logged.
- Trends now renders labeled non-color calorie and macro series plus exact tables, planned-versus-confirmed and record-completeness tables, and collapsed advanced source/matrix sections. Macro mode selects between two already-built representations; it does not calculate in the browser. Goal-unauthorized viewers get a clear unavailable state.
- The Trends page no longer invokes the all-history/per-row intake service or Overview recommendation path. Other existing views keep their established behavior.

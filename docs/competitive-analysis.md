# Competitive analysis implications

Research against official Mealie, Tandoor, RecipeSage, Paprika, AnyList, and Samsung Food product documentation found a consistent household pattern: capture/import, organization, cooking, planning, and shopping are a connected workflow. The future architecture therefore keeps shared household content separate from profile-level preferences.

The first foundation adopts the parts that need to be true before recipe work: explicit local household context, editable personal unit/temperature preferences, a review-first future capture direction, and no misleading claim that profiles are accounts. It deliberately does not reproduce any competitor’s branded copy, visual design, or proprietary implementation.
